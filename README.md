# ТЗ на Telegram‑бота kick

## 1. Цель и общая идея

Бот — это нейтральный инструмент для регулярной практики осознавания.По сути своей интеллектуальный таймер, который присылает сообщения по времени и может задавать тематические вопросы.

Он нужен для двух вещей:
1. Сырой дневник — честно «сливать» мысли, эмоции, события без фильтров.
2. Структурированная практика (по ДЗ) — несколько раз в день проходить через одинаковую последовательность шагов осознавания.

Бот:
- не оценивает,
- не утешает,
- не даёт советов,
- не интерпретирует,
- только задаёт вопросы, фиксирует и показывает накопленное.

Записи сохраняются в базу данных отдельно для каждого пользователя с полями "дата (date)", "содержание (context)", "отношение (attitude)", "форма согласия (form)", "реакция тела (body)", "изменения (response)". Собственно каждое поле - это каждый вопрос и плюс дата.

## 2. Основной функционал

### 2.1. Напоминания

- Пользователь может настроить напоминания в свободной форме:
  - «Напоминай мне с 9 до 22 каждые 2 часа»
  - «В будни в 10:00, 13:00, 17:00 и 21:00»
- В указанные моменты бот присылает короткое сообщение, например:
  - «Готов записать наблюдение?». И кнопки:
    1 "Да, пошагово"
    2 "Да"
    3 "Нет"
  1 Предоставить сисок тем/вопросов пошагово:
    - Обратите внимание на содержание в поле вашего внимания (внутреннее или внешнее);
    - Осознайте своё отношение к этому содержанию - обратите внимание на тело, на ваш баланс расслабления и напряжения по поводу этого содержания;
    - Подберите свою форму выражения этого отношения, вербализовав его, используя формы согласия и отрицания. Да-принимающее, Нет-принимающее, Да-отрицающее, Нет-отрицающее.
    - Озвучьте для себя и обратите внимание соответствует ли то, что вы осознали телесной реакции? 
    - Обратите внимание, что будет происходить с вами после осознания;
    На каждое сообщение спрашивать:
    - "Следующий шаг"
    После последнего вопроса показать кнопку:
    - "Подытожить"
    - "Сохранить"
    При получении сообщения кнопки удалять и показать заново.

  2 Предоставить ориентировочный сисок тем/вопросов и далее ожидать ввода пользователя голосом или текстом:
    - Обратите внимание на содержание в поле вашего внимания (внутреннее или внешнее);
    - Осознайте своё отношение к этому содержанию - обратите внимание на тело, на ваш баланс расслабления и напряжения по поводу этого содержания;
    - Подберите свою форму выражения этого отношения, вербализовав его, используя формы согласия и отрицания. Да-принимающее, Нет-принимающее, Да-отрицающее, Нет-отрицающее.
    - Озвучьте для себя и обратите внимание соответствует ли то, что вы осознали телесной реакции? 
    - Обратите внимание, что будет происходить с вами после осознания;
    На каждое сообщение спрашивать:
    - "Подытожить"
    - "Сохранить"
    При получении сообщения кнопки удалять и показать заново.
  3 Завершить разговор сохранив в базу данных отказ.

### 2.2. Записи: голос и текст

- Пользователь может:
  - отправлять голосовые сообщения,
  - писать текст.
- Бот всегда:
  - принимает голос / текст,
  - возвращает пользователю распознанный/полученный текст для проверки,
  - даёт опции:
    - «Всё ок»
    - «Уточнить/добавить»
    - "Задать наводящие вопросы по теме"
    - ""
    - «Завершить»

## 3. Работа c накопленными записями

Бот должен уметь по запросу пользователя показывать историю:

Примеры запросов:
- «Покажи результаты»
    - "json"
    - "txt"

Бот сформирует текстовый файл и пришлет в чат.

Формат выдачи (для каждой записи):

- Дата и время.
- Тип записи: «сырой дневник» или «структурированная практика».
- Краткая суммаризация (если есть).
- Полный текст.

Пример:

2025-11-20 10:15
Содержание: ...
Отношение: ...
Форма согласия: ...
Реакция тела: ...
Изменения: ...

2025-11-20 10:15
...

Если формат json - то на каждую дату соответствующие поля.

Так бот умеет очищать историю - тоесть полностью удалять все записи.

## 4. Стиль общения бота

- Максимально нейтрально.
- При суммаризации и подытоживании опираться только на слова пользователя.
- Без смайликов, без мотивации, без оценок и советов.
- Примеры допустимых фраз:
  - «Готов записать наблюдение?»
  - «Отправь голосовое сообщение или напиши текст.»
  - «Перейти к следующему шагу?»
  - «Завершить запись?»
  - «Запись сохранена.»
  - «Вот записи за выбранный период.»
